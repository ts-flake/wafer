*A RNN with its output mapped through a dense layer.

Input: x, shape (N, L, D_in) or (N, D_in) Output: outp, shape (N, L, D_out) or (N, D_out)*

nh
no

LSTM

 LSTM (ni, nh, num_layers=1,
       actn:Union[str,torch.nn.modules.module.Module]='tanh',
       gate_actn:Union[str,torch.nn.modules.module.Module]='sigmoid',
       dropout=0.0, unit_forget_bias=True,
       init_gain=np.float64(0.5773502691896258), recurrent_init_gain=1.0,
       return_seq=False)

*Custom LSTM, allowing for different activations. Assuming batch_first=True and bidirectional=False.

Inputs: x, h0_c0; x: shape (N, L, D_in) or (N, D_in). h0_c0: (h0, c0), Union[list, tuple], optional, default zeros, each of shape (num_layers, N, hidden_size).

Outputs: output, (hn, cn); output: shape (N, L, hidden_size) or (N, hidden_size), outputs of the last layer for each token. hn: shape (num_layers, N, hidden_size), final hidden state. cn: shape (num_layers, N, hidden_size), final cell state.*

SimpleICRNN

 SimpleICRNN (ni:int, nh:int, no:int,
              actn:Union[str,torch.nn.modules.module.Module]='relu',
              out_actn:Union[str,torch.nn.modules.module.Module]=None,
              expand_inp:bool=True, return_seq:bool=False)

*Simple input-convex RNN. h = f(Wh_ + Ux + b); y = g(Vh + b), all weights are non-negative and activations are convex and non-decreasing. Weights are initialized as in np-RNN.

Inputs: x, shape (N,L,D_in) Outputs: outputs, shape (N,L,D_out)*

Type Default Details
ni int input size
nh int hidden size
no int output size
actn Union relu hidden activation
out_actn Union None output activation
expand_inp bool True expand the input to [x, -x]
return_seq bool False return the whole seuqnece

ICRNN

 ICRNN (ni:int, nh:int, no:int,
        actn:Union[str,torch.nn.modules.module.Module]='relu',
        out_actn:Union[str,torch.nn.modules.module.Module]=None,
        expand_inp:bool=True, init_gain:float=1.0,
        recurrent_init_gain:float=1.0, return_seq:bool=False)

*Input-convex RNN. Refer to Optimal Control Via Neural Networks: A Convex Approach. The recurrent weight matrix is initialized as Identity * recurrent_init_gain.

Inputs: x, shape (N,L,D_in) Outputs: outputs, shape (N,L,D_out)*

Type Default Details
ni int input size
nh int hidden size
no int output size
actn Union relu hidden activation
out_actn Union None output activation
expand_inp bool True expand the input to [x, -x]
init_gain float 1.0 weight initialization gain
recurrent_init_gain float 1.0 recurrent weight initialization gain
return_seq bool False return the whole seuqnece
